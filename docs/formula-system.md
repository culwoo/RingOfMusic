# 디자인 공식·취향 축적·제작 스펙 시스템 안내

버전: 1.0
작성일: 2026-07-04
근거: docs/productization-strategy.md의 Phase 1 항목 구현 결과

## 이번에 추가된 것

이번 구현은 전략 문서의 핵심 진단 두 가지를 코드로 해소하였다. 디자인 공식의 소유권을 개발자 코드에서 디자이너 데이터로 옮겼고, 스케치에서 제작으로 가는 다리를 놓았다.

### 1. 디자인 공식 시스템 (L1)

`sketchPrompt.ts`에 하드코딩되어 있던 소재·변형 테마·금지 목록·브랜드 문구가 서버 저장 데이터로 승격되었다. 디자이너는 앱의 "디자인 공식" 패널에서 직접 수정한다. 저장할 때마다 버전이 1씩 올라가고 이전 버전은 이력에 보관된다. 스케치 캐시 키에 공식 버전이 포함되므로, 어떤 시안이 어떤 공식에서 나왔는지 항상 추적된다.

- 저장 위치: `data/formula.json` (첫 기동 시 기본 공식 v1 자동 시드)
- API: `GET/PUT /api/formula`, `GET /api/formula/history`
- 코드: `server/formula-store.mjs`, `server/formula-defaults.mjs`, `src/components/FormulaPanel.tsx`

### 2. 취향 피드백 축적 (디자이너 락인 장치)

스케치 하트, 최종 선택, 선택 이유 태그가 전부 서버에 이벤트로 쌓인다. 하트를 누르면 카드 아래 이유 태그(비율 좋음, 스톤 과함 등)가 나타난다. 공식 패널의 "취향 인사이트"가 방향별·소재별 선호와 태그 통계를 보여준다. 이 데이터는 공식 개정의 근거이자, 향후 LoRA 학습 데이터 선별 기준이 된다. 전략 문서의 "공식 추출 워크숍 2단계(비교 선택)"가 제품 기능으로 내장된 것이다.

- 저장 위치: `data/feedback.json`
- API: `POST /api/feedback`, `GET /api/feedback/insights`
- 코드: `server/feedback-store.mjs`, `src/lib/feedbackApi.ts`

### 3. 제작 스펙 내보내기 (트랙 B)

우측 인스펙터의 "제작 스펙 내보내기" 버튼이 두 파일을 내려준다. 스펙 JSON은 호수→내경(mm), 밴드 폭·두께, 릴리프 깊이, 스톤 위치(각도)를 담는다. 파형 CSV는 선택 구간을 256샘플로 리샘플링해 각도별 진폭과 릴리프 깊이(mm)를 제공한다. 그래스호퍼 정의에 바로 연결하는 용도이다. 슬라이더→mm 변환식은 스펙의 mappingNotes에 명시되며, 최소 벽두께 위반 시 경고가 포함된다.

- 코드: `src/lib/spec.ts` (기준 치수는 공식의 "제작 기준 치수"에서 편집)

### 4. 원본 파일 영구 보존 (버그 수정)

기존에는 업로드 원본이 파형 분석 직후 삭제되었다. 이제 `data/media/`로 이동·보존되고 `/media/...` URL로 서빙된다. 프로젝트를 다시 열어도 저장된 원본으로 재생이 가능하다.

## 실행과 검증

실행 방법은 기존과 같다(`npm run dev`). 검증 결과는 다음과 같다.

- 서버 테스트 13건 통과 (`npm test`): 공식 저장소 4건, 피드백 저장소 3건 신규 포함
- `tsc -b` 타입체크 통과, `vite build` 프로덕션 빌드 성공
- API 스모크 테스트 통과: 공식 시드→수정 저장(v2)→이력 보관, 잘못된 입력 400 거부, 피드백 기록→인사이트 집계

## 운영 메모

- `data/formula.json`, `data/feedback.json`, `data/media/`는 런타임 자산이므로 .gitignore에 추가되어 있다. 백업 대상에는 반드시 포함할 것 (고객 원본과 디자이너 공식이 여기 있다).
- `backup/baseline-20260704.tar.gz`는 이번 고도화 직전 상태, `backup/after-upgrades-20260704.tar.gz`는 직후 상태의 스냅샷이다.
- 공식을 수정하면 캐시 키가 바뀌어 기존 스케치 캐시를 재사용하지 않는다. 의도된 동작이다(공식이 다르면 다른 결과여야 한다).
- 다음 단계는 전략 문서 6장의 Phase 0(디자이너 공식 추출 워크숍)과 Phase 2(B2C 접점)이다. 워크숍 결과를 공식 패널에 입력하는 순간부터 이 시스템이 실제 디자이너의 공식을 담기 시작한다.
