# Ring of Music - 프로젝트 개요

버전: 0.3 고객 업로드/구간 선택 흐름 반영  
작성일: 2026-05-09

## 한 줄 설명

Ring of Music은 영상 또는 음성 파일 안에 담긴 의미 있는 소리의 순간을 반지 디자인 콘셉트, 다중 2D 스케치, 향후 3D 모델 프리뷰로 변환하는 디자이너 중심 주얼리 제작 플랫폼이다.

## 핵심 아이디어

사람들은 주얼리와 액세서리에 개인적인 의미와 이야기를 담고 싶어 한다. 예를 들어 아이의 첫 말, 결혼식 서약, 사랑하는 사람의 목소리, 고백, 노래의 한 구간, 반려동물의 소리, 돌아가신 가족의 음성 같은 순간은 단순한 장식보다 강한 감정적 가치를 가진다.

현재 이런 이야기는 카카오톡, 이메일, 클라우드 링크, 영상 파일, 텍스트 설명 등으로 디자이너에게 전달된다. 고객이 직접 mp4를 mp3로 변환하거나, 특정 구간을 잘라 보내거나, 파형 이미지를 만드는 것은 번거롭다. Ring of Music은 고객이 가진 원본 파일에서 음성을 추출하고, 고객 또는 디자이너가 그 소리를 직접 들으며 파형 구간을 선택한 뒤, 선택된 소리 데이터를 반지 디자인으로 해석할 수 있게 만든다.

고객 입력 방식은 두 가지를 모두 지원해야 한다. 어떤 고객은 "음원은 이거고, 반지가 예쁘면 돼요"처럼 파일과 분위기만 맡기고 싶어 한다. 반대로 어떤 고객은 "몇 분 몇 초부터 몇 분 몇 초까지, 이 부분이어야 해요"처럼 정확한 순간을 직접 지정하고 싶어 한다. 따라서 고객용 웹은 업로드만 하고 디자이너에게 선택을 맡기는 흐름과, 업로드 후 파형을 들으며 직접 구간을 선택하는 흐름을 모두 제공해야 한다.

## 제품 포지셔닝

Ring of Music은 고객이 직접 3D 모델링까지 수행하는 셀프서비스 반지 제작기가 아니다. 첫 번째 제품은 고객의 영상/음원/스토리를 받은 이후, 디자이너와 제작팀이 이를 반지 디자인으로 바꾸는 과정에 집중한다.

고객용 웹은 업로드, 구간 선택, 상담, 로그인, 제작 이력 확인을 쉽게 만드는 접점이다. 하지만 제품의 핵심 가치는 디자이너용 제작 워크스페이스에 있다.

공식 파일 접수는 카카오톡 같은 국내 메신저나 개별 채팅 서비스에 의존하지 않고 웹 업로드로 처리한다. 이유는 글로벌 고객이 같은 방식으로 접근할 수 있어야 하고, 영상/음원 파일은 용량이 커서 메신저 전송 제한이나 압축 문제가 생기기 쉽기 때문이다. 또한 웹 업로드를 사용해야 원본 파일, 선택 구간, 상담 기록, 디자인 시안, 제작 이력을 하나의 프로젝트 안에 안정적으로 보존할 수 있다.

시장조사 결과, 단순한 사운드웨이브 반지와 음파 각인 주얼리는 이미 저가 Etsy 상품부터 프리미엄 수공예 브랜드까지 존재한다. 따라서 Ring of Music은 "소리 파형을 새긴 반지" 자체로 차별화하기 어렵다. 차별점은 미디어 업로드, 음성 추출, 파형 구간 선택, 감정적 해석, 여러 디자인 방향의 스케치 생성, 비교, 그리고 향후 3D/제작 연계까지 이어지는 디자이너 워크플로우가 되어야 한다.

## 사용자

### 고객

고객은 자신의 이야기가 담긴 반지를 만들고 싶지만, 음원 편집, 파형 디자인, CAD 모델링, 제조 제약을 이해할 필요는 없다.

고객의 주요 과업:

- mp4, mov, wav, mp3 등 형식 변환 없이 음성이 포함된 파일을 업로드한다.
- 어떤 소리나 구간이 중요한지 설명한다.
- 업로드 후 파형을 직접 들으며 원하는 구간을 선택한다.
- 구간을 직접 고르기 어렵거나 디자인 감각을 맡기고 싶으면, 파일만 업로드하고 디자이너에게 선택을 맡긴다.
- 제작팀과 1:1 상담을 진행한다.
- 로그인 후 이전에 제작했거나 진행 중인 반지 프로젝트를 확인한다.
- 개별 반지 페이지에서 업로드한 영상/음원, 파형, 선택 구간, 디자인 진행 상황을 확인한다.

### 디자이너

디자이너는 고객의 자료를 받아 주얼리 콘셉트로 해석하는 사람이다. 디자이너에게 필요한 것은 고객 대신 파일을 정리하고, 소리를 듣고, 의미 있는 구간을 찾고, 그 구간을 시각적/물리적 디자인으로 바꾸는 도구다.

디자이너의 주요 과업:

- 고객이 업로드한 영상 또는 음성 파일을 검토한다.
- 파일에서 음성 트랙을 추출하고 파형을 확인한다.
- 고객이 선택한 구간이 있으면 이를 확인하고, 필요하면 더 정교하게 다듬는다.
- 고객이 구간 선택을 맡긴 경우 직접 듣고 재생 위치를 조정하며 디자인에 적합한 구간을 선택한다.
- 하나의 디자인 방향에 대해 여러 장의 2D 스케치를 생성한다.
- 같은 파형 구간에서 여러 디자인 방향을 생성한다.
- 스케치들을 비교해 3D 모델링 전에 가장 가능성 높은 방향을 선택한다.
- 향후 3D 프리뷰에서 비율, 두께, 표면 디테일, 착용감을 확인한다.
- 각 콘셉트별로 원본 파일, 타임스탬프, 선택 파형, 고객 메모, 스케치, 디자인 파라미터를 보존한다.

## 고객용 웹 경험

고객용 웹은 네 영역으로 구성된다.

1. 랜딩페이지
   - 개인의 소리와 이야기를 반지로 만든다는 콘셉트를 전달한다.
   - 주요 버튼은 "반지 만들러 가기"이다.

2. 고객 대시보드
   - 진행 중이거나 이전에 제작한 반지 프로젝트를 보여준다.
   - 한 사람이 여러 번 서비스를 사용할 수 있으므로, 로그인 한 번으로 이력을 확인할 수 있어야 한다.

3. 업로드 및 구간 선택 플로우
   - 고객은 음성이 포함된 영상 또는 음성 파일을 웹에서 업로드한다.
   - 시스템은 파일에서 음성을 추출하고 파형을 보여준다.
   - 고객은 재생, 일시정지, 탐색을 하며 원하는 구간을 선택할 수 있다.
   - 고객은 "직접 선택한 구간으로 진행" 또는 "디자이너에게 구간 선택 맡기기"를 선택할 수 있다.
   - 영상 업로드의 경우 영상 화면과 음성 파형이 함께 동기화되어야 한다.

4. 개별 반지 프로젝트 페이지
   - 업로드한 영상 또는 음성 파일을 보여준다.
   - 추출된 파형을 보여준다.
   - 고객이 직접 선택한 파형 구간 또는 디자이너가 선택한 파형 구간을 확인할 수 있다.
   - 영상 업로드의 경우 영상과 음성이 함께 재생된다.
   - 제작팀과 1:1 상담할 수 있다.

## 디자이너 워크스페이스

디자이너 워크스페이스가 첫 번째 구현 우선순위다. 시작은 웹 기반 스튜디오와 서버 기반 미디어 처리로 가는 것이 적절하다. 추후 로컬 파일 처리, 오프라인 작업, Rhino/MatrixGold/Blender 같은 툴과의 직접 연동이 필요해지면 데스크탑 앱으로 확장할 수 있다.

초기 디자이너 플로우:

1. 디자이너가 영상 또는 음성 파일을 수동 업로드한다.
2. 시스템이 음성 트랙을 추출한다.
3. 인터랙티브 파형을 렌더링한다.
4. 디자이너가 듣기, 일시정지, 탐색, 구간 선택을 수행한다.
5. 선택된 파형으로 하나의 디자인 방향에 대한 여러 2D 스케치를 생성한다.
6. 여러 스케치를 비교하고 유망한 콘셉트를 고른다.
7. 이후 여러 디자인 방향을 생성하고, 각 방향마다 여러 스케치를 비교한다.
8. 선택된 콘셉트에 대해 3D 프리뷰를 생성한다.

## 파형의 디자인 변환 방향

선택된 파형은 다음과 같은 디자인 언어로 변환될 수 있다.

- 원형 파형: 소리 데이터를 반지 둘레를 따라 감싼다.
- 밴드 릴리프: 파형의 진폭을 반지 표면의 돌출 또는 음각 디테일로 바꾼다.
- 외곽선 실루엣: 파형이 반지의 외부 윤곽을 만든다.
- 지형형 릴리프: 파형의 강약을 지형처럼 높낮이가 있는 표면으로 만든다.
- 이너 밴드 각인: 소리 패턴을 착용자만 아는 안쪽 각인으로 만든다.
- 시그넷 표면 패턴: 파형을 시그넷 링 상단의 음각/양각 패턴으로 만든다.
- 네거티브 스페이스: 파형을 절개, 홈, 빈 공간의 패턴으로 만든다.
- 스톤 리듬 배치: 파형의 피크를 보석의 위치, 크기, 간격에 반영한다.
- 듀얼 파형: 두 사람의 목소리나 두 구간의 소리를 겹쳐 관계성을 표현한다.

중요한 점은 파형을 그대로 복사하는 것이 아니라, 디자이너 공식과 제약을 통해 착용 가능하고 제조 가능한 디자인으로 해석하는 것이다.

## MVP 전략

MVP는 고객 랜딩페이지보다 디자이너 도구부터 시작해야 한다. 가장 큰 제품 리스크는 "파형이 실제로 매력적인 반지 디자인으로 바뀔 수 있는가"이기 때문이다.

권장 개발 순서:

1. 디자이너가 영상 또는 음성 파일을 수동 업로드한다.
2. 서버에서 음성 추출과 정규화된 파형 생성을 수행한다.
3. 웹에서 파형 재생, 탐색, 구간 선택을 구현한다.
4. 하나의 디자인 방향에 대해 여러 2D 스케치를 생성한다.
5. 여러 디자인 방향과 각 방향별 스케치 세트를 생성한다.
6. 프로젝트별 원본 파일, 선택 구간, 파형, 고객 메모, 스케치 결과를 보존한다.
7. 선택된 디자인에 대해 가벼운 3D 프리뷰를 만든다.
8. 고객 랜딩페이지, 대시보드, 웹 업로드, 선택형 구간 지정, 상담 기능을 붙인다.
9. 고객 프로젝트와 디자이너 워크스페이스를 연결한다.

## 기술 방향

고객용 제품은 접근성, 글로벌 사용, 로그인, 업로드, 제작 이력, 상담을 위해 웹 기반이어야 한다.

디자이너용 제품도 초기에는 웹 기반 스튜디오가 적합하다. 단, 영상/음성 처리 자체는 브라우저에 모두 맡기기보다 서버에서 FFmpeg 같은 미디어 처리 파이프라인으로 수행하는 것이 안정적이다. 이렇게 하면 이후 고객 업로드, 디자이너 검토, AI 스케치 생성, 프로젝트 관리, 상담 기능을 하나의 백엔드에서 공유할 수 있다.

브라우저는 파형 인터랙션, 구간 선택, 스케치 비교, 가벼운 3D 프리뷰에 적합하다. 데스크탑 앱은 다음 조건이 생겼을 때 고려한다.

- 대용량 원본 파일을 로컬에서만 처리해야 한다.
- 개인정보나 저작권 문제 때문에 업로드 없는 로컬 작업이 필요하다.
- Rhino, MatrixGold, Blender, ZBrush 같은 기존 모델링 툴과 직접 연동해야 한다.
- 3D 생성이나 렌더링이 GPU 중심으로 무거워진다.
- 디자이너가 오프라인 스튜디오 환경에서 작업해야 한다.

현실적인 전략은 웹으로 시작하고, 나중에 필요하면 같은 UI를 Tauri/Electron으로 감싸거나 로컬 미디어/CAD 브릿지를 추가하는 것이다.

## 핵심 리스크

- 감정 번역 리스크: 생성된 반지가 선택된 소리와 의미 있게 연결되어야 한다.
- 미적 리스크: 원본 파형을 그대로 쓰면 임의적이거나 투박해 보일 수 있다.
- 제조 리스크: 얇은 피크, 날카로운 굴곡, 약한 구조가 생길 수 있다.
- 프리뷰-실물 차이 리스크: 2D 스케치와 3D 프리뷰가 실제 제작 불가능한 디테일을 약속하면 안 된다.
- 워크플로우 리스크: 고객에게 모델링을 강요하지 않고 디자이너를 도와야 한다.
- 법적/신뢰 리스크: 업로드 파일에는 저작권 음악, 사적인 목소리, 아동 음성, 민감한 추억이 포함될 수 있다.
- 상품화 리스크: 단순 음파 각인은 이미 존재하므로 디자인 품질과 제작 워크플로우로 차별화해야 한다.
- 렌더링 리스크: 낮은 품질의 실시간 3D 프리뷰는 금속과 보석을 싸구려처럼 보이게 만들 수 있으므로 2D 스케치와 정적 렌더 품질을 먼저 확보해야 한다.

- 디자이너는 이 도구를 내부 제작 도구, 고객 승인 도구, 리드 생성 도구 중 무엇으로 가장 가치 있게 느끼는가?
- 커플, 웨딩, 추모, 부모-자녀, 팬덤, 아티스트 굿즈 중 어떤 세그먼트가 가장 강한 구매 의도를 보이는가?
- 고객은 원본 파형의 정확성을 더 중시하는가, 디자이너가 해석한 아름다운 추상화를 더 중시하는가?
- 제작 CAD로 넘어가기 전에 어느 수준의 3D 프리뷰가 충분한가?
